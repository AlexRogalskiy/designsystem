Typography

Heading Strategy

To allow for accessibility while maintaining flexibility in design, we have chosen to use the semantically correct header tags (h1, h2, h3, etc.) combined with styling classes. This means that in a situation where structure and design diverge, it is possible to override the default header.

Example code

<h1 class="h1">First Heading</h1> <p class="lead-paragraph">Some leading text</p> <h2 class="h3">Second Heading</h2> <p>This one is an h2 with h3 styling.</p>

Check box

Used when there are several possible options, have a regular and an -inline variant

Example code

<input class="input-field" type="checkbox" name="unicornfood" value="apples" id="unicornfood-1" checked> <label id="unicornfood-1-label" class="check-box -inline" for="unicornfood-1">Apples</label>

Radio button

A standard type of radio button. They have a regular and an -inline variant

Example code

<input class="radio-input" type="radio" name="unicornwings" value="yes" id="unicornwings-1" checked> <label id="unicornwings-1-label" class="radio-button -inline" for="unicornwings-1">Yes</label>

Radio switch

Intended for cases with only two mutually exclusive options

Example code

<input class="radio-input" type="radio" name="unicornslike" value="yes" id="unicornslike-1" checked> <label id="unicornslike-1-label" class="radio-switch" for="unicornslike-1">Ja</label>

Radio block

Radio blocks are perfect when there is a great need to present contextual relevant information to the user regarding the current choice without overwhelming them

Jeg skal forsikre en ny bil og overføre bonus fra en forsikring som er avsluttet, eller som jeg skal avslutte.
