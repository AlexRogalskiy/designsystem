Hawaii Layout System

Layouts are composed of one or more sections wrapping content containers.

  • Do not put any more classes on them (except modifiers)
  • Think separation of concerns, a section contains other things so they should not have any other responsibility
  • It should be a flexible system, but if you feel the need for more flags, consider making that behaviour part of a component instead

.section-wrapper

  • Responsible to uphold the standard layout rules concerning app gutters
  • Wrapper for content containers
  • Provides seamless stacking
  • Does not need to be of the HTML5 section tag, but could be a good semantic structure
Modifier Description
--bg-sand Setting background color to Sand-25%
--bg-grey Setting background color to Grey-light-bg

.content-container

  • Manages the direct layout of content, text or components
  • Default centering of content
  • Provide horizontal spacing when stacked
  • Depending on breakpoints for dynamic vertical margins
Modifier Description
--sm Small container
--lg Large container
--text-left Align text to the left, instead of the default centering
--no-space Removes default spacing
